Using Genetic Algorithms to Find the Best Path

James Gaiser / October 1, 2023

Current State of Pathfinding

Pathfinding, that is the task of trying to determine a path between a starting point and an end point, has had a known solution for over 50 years at this point: A*. It works by looking at all its possible options that take it the closest to the end goal. With a good heuristic, it is guaranteed to find an optimal path and is commonly used today.

However, a key weakness of A* is its high computational cost; it must consider a large number of nodes to find its path. This can turn out to be a big problem, considering one of A*â€™s main usage is in video games where performance can heavily influence the quality of a program.

Introducing Genetic Algorithms

Genetic algorithms have promise to help optimize path finding. They work by encoding different behaviors into data structures that act like genes in an animal and are selectively pruned over generations akin to natural selection in real species. From what I have seen online, there are two approaches of implementing genetic algorithms that would be helpful to look at.

The first kind is a pure genetic algorithm approach where you simply give a genetic algorithm a map and let it find the best path. They can handle a wide variety of starting conditions and they tend to get better at finding a path with each successive generation. However, these algorithms appear to have a poor ability to generalize; that is if you were to take the genes of a model already trained on one map and placed it on another, they would need to train again to recover the same level of fitness.

Clearly the problem lies in how we are using genetic algorithms. Essentially we trained a genetic algorithm to solve a single map when in actuality we want it to be able to find a path in any solvable map. A better idea would be to use a genetic algorithm informed approach where we still utilize traditional pathfinding algorithms like A* or Best-First-Search.

In a paper by Santos et al, genetic algorithms were suggested as a way to simplify the map for traditional algorithms to work on. This was achieved by training the model on recognizing patterns in maps and having it generate a smaller path through these patterns, decreasing the amount of nodes that need to be searched.

Key Takeaways

With careful implementation, genetic algorithms can be extremely helpful in optimizing path finding algorithms. They operate best when they aid the search of already established pathfinding algorithms and can create considerable performance increases.
